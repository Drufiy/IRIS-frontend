What IRIS Actually Does

Not a chat wrapper. A working system interface.

Built around real desktop control, memory, browser automation, and voice interaction instead of a single prompt box.

Launch and Control

Open apps, focus windows, run shell actions, and move through tasks without touching the keyboard first.

Browser Automation

Navigate sites, fill forms, extract information, and coordinate web tasks from a single voice-driven flow.

Memory and Context

IRIS stores useful interaction context locally so it can respond with continuity instead of starting from zero.

Speech In and Out

Groq Whisper handles speech recognition and ElevenLabs delivers spoken replies with low-friction feedback.

Task Routing

Requests are planned, routed, and executed through the agent pipeline rather than treated like one-shot chat.

Local or Remote

IRIS supports both direct local API configuration and a shared remote backend mode, depending on how you want to run it.

How It Works

From wake word to execution in one visible chain.

01

Wake

IRIS starts from voice. A wake word or direct request pushes the system into an active state.

02

Understand

Speech becomes text, context is injected, and the request is framed for actual execution.

03

Plan

The agent layer decomposes the task, selects the route, and keeps the system aligned with intent.

04

Execute

Files, browser actions, shell tasks, reminders, and responses all move through one action system.

05

Respond

IRIS answers with speech and state feedback so the interaction feels like a running operating layer.

Live State Language

Minimal interface. Clear signal.

The IRIS identity comes from state, not clutter. Idle stays quiet. Listening sharpens. Action glows. Approval becomes explicit.

IDLE

White, static, waiting for intent.

LISTENING

Blue, precise, attentive.

ACTING

Green, forward, executable.

APPROVAL

Warned and deliberate when risk matters.

Safety

System control without silent overreach.

Action Classification

IRIS separates safe, warned, and dangerous actions before execution instead of treating everything equally.

Approval Gates

Destructive or higher-risk actions require explicit user approval through the interface before they run.

Local Memory

Conversation memory and context storage stay local, while cloud providers and proxy mode remain explicit and configurable.
