Capabilities

IRIS is built to move across real system work.

The backend already spans speech, agents, memory, browser automation, reminders, shell actions, and desktop control. This page turns those raw pieces into a clearer product surface.

Speech Groq Whisper + ElevenLabs
Routing DeepSeek-backed planning and execution
Actions Desktop, browser, reminders, tasks, files
Feedback Tauri overlay and live visible state

Capability Groups

One assistant surface, several execution layers.

Input

Voice-first command intake

Wake word support, ASR transcription, interruption handling, and a loop designed around spoken interaction rather than typed prompts.

  • Wake word flow
  • Groq Whisper transcription
  • Conversational turn handling
Reasoning

Task routing and agent execution

IRIS routes requests into planning, execution, and browser/code-oriented paths so the answer can become action instead of staying text-only.

  • DeepSeek model routing
  • Planner and executor agents
  • Action-oriented responses
Control

Desktop and browser action layer

Launch apps, navigate the browser, manage reminders, run shell tasks, and coordinate common workflows through one system surface.

  • File and shell actions
  • Browser automation
  • Task and reminder flows
Memory

Context that carries forward

IRIS stores useful context and retrieves it later so the assistant behaves more like a running system and less like a blank chat every time.

  • Short-term context
  • Long-term memory store
  • Knowledge graph direction
Output

Speech and visible state feedback

Responses come back through speech and UI state together so the assistant stays legible while it is listening, acting, or waiting for approval.

  • ElevenLabs TTS
  • Tauri overlay feedback
  • Interactive approval prompts
Modes

Local and remote operating modes

Run IRIS against your own keys locally or against a shared backend proxy depending on how you want to deploy the assistant.

  • Local provider mode
  • Remote proxy mode
  • Explicit provider configuration
