Common Questions

The short version.

Is IRIS already a working desktop app?

Not yet. The backend systems exist, and the website now defines the product language, but the actual Tauri desktop app shell still needs to be implemented.

Can I use IRIS today?

Yes, through the open-source repos and source-first setup path. The packaged desktop download experience is not the primary access path yet.

Will there be both Windows and macOS versions?

Yes. The goal is one shared product with platform-specific adaptations, not two unrelated interfaces.

What is already real in the backend?

Speech input and output, agent routing, browser and shell actions, memory systems, safety gates, and a Tauri-oriented overlay direction are already part of the broader IRIS effort.

Why finish the website first?

Because the website locks the product story, visual language, and interaction expectations before the real app shell starts. That makes desktop implementation faster and more coherent afterward.

How do I follow release progress?

Use the changelog, the repos, and the waitlist page. Those are the cleanest surfaces until the desktop installer flow is live.
